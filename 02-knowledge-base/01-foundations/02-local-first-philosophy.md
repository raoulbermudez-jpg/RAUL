## Propósito

Documenta los principios que guían las decisiones arquitectónicas del PKA /RAUL/: por qué se eligió un sistema basado en archivos locales Markdown en lugar de herramientas SaaS, y las implicaciones prácticas de ese enfoque.

## Estado

> **Stub** — pendiente de expansión.

## Principios núcleo

1. **Local-first:** el sistema de archivos es la base de datos. El conocimiento existe independientemente de cualquier herramienta.
2. **Vendor-neutral:** ningún agente, flujo o documento crea dependencia a un formato o SaaS propietario como verdad principal.
3. **Multi-LLM:** cualquier modelo (Claude, GPT, Gemini, Perplexity) puede operar sobre el sistema sin instalación especial.
4. **App-less:** Notion, Obsidian, Roam y equivalentes pueden ser UI complementaria, pero nunca SSOT.
