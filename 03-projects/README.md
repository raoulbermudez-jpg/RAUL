# 03-projects/

Trabajo operativo por dominio y proyecto.

## Organización

Dominio → proyecto → etapas numeradas.

- `genteca/` · `finca-ganaderia/` · `plenus-metabolica/` · `teca-teak4food/` · `marca-personal-raoul/`
- `research-generic/` — investigación sin dominio fijo.
- `sandbox-and-experiments/` — experimentos y pruebas.

## Convenciones

- Carpeta de proyecto: `YYYY-MM_slug-descriptivo/`.
- Etapas: `00-brief/` → `01-strategy-and-design/` → `02-production/` → `03-review-and-release/` → `04-published-and-hand-off/`.
- Cada proyecto tiene `README.md` con objetivo, stakeholders, enlaces a KB y estado.

## Regla

Aquí pueden convivir Markdown, código y binarios. Git versiona solo texto/código; los binarios pesados están gitignored (ver `.gitignore`).
